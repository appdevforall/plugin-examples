AI Core Plugin

The shared LLM inference engine for Code on the Go (CoGo), providing on-device (llama.cpp) and Google Gemini backends to the other AI plugins.

1. Executive Overview

AI Core is a headless plugin — it contributes no screens of its own. Its job is to register a single LlmInferenceService that every other AI plugin discovers and calls at runtime. Install AI Core first, then any plugin that depends on it:

  • AI Assistant — the conversational agent and its tool-loop.
  • Code Suggestions, Speech to Text, Vector Search — other consumers of the inference service.

Plugins are bridged through a shared service registry (SharedServices): AI Core publishes the service on activation and removes it on deactivation, so consumers resolve it dynamically without a compile-time dependency.

2. Core Functionality

  • Dual inference backends — a fully offline, on-device .gguf model via bundled llama.cpp, or Google Gemini in the cloud, selectable from AI Assistant's settings.
  • Streaming generation — token-by-token streaming with cancellation, so a running response can be stopped promptly.
  • Tool/function calling — the Gemini backend supports tool calls, driving the AI Assistant agent loop.
  • Model validation — embedding-only .gguf files are detected and rejected with a clear error instead of crashing the runtime.
  • Storage-safe model loading — models chosen via the Storage Access Framework (content://) are streamed to a private cache path before use.

3. Technical Architecture

consumer plugins (AI Assistant, …)
        │  discover via SharedServices
        ▼
ai-core  (LlmInferenceService)
        ├── LocalLlmBackend  ──► llama-impl ──► llama.cpp (native, JNI)
        └── GeminiBackend    ──► Google GenAI SDK (HTTPS)
  • llama-impl / llama-api — Android modules wrapping the native llama.cpp build via JNI; the prebuilt .aar ships the native .so libraries (declared with the native.code permission).
  • Network — only the Gemini backend performs network calls; the plugin holds network.access and INTERNET solely for that. The Gemini API key is sent as an x-goog-api-key header over HTTPS, never in a URL and never logged.
  • Lifecycle — on dispose the in-flight generation is cancelled, the native model is unloaded, and all backend coroutine scopes are cancelled so nothing leaks when the plugin is unloaded.

4. Usage

Build

A normal build needs no submodule or NDK — the llama.cpp library ships prebuilt in the .aar.

cd ai-core && ./gradlew assemblePlugin

Output lands at ai-core/build/plugin/ai-core.cgp. See BUILDING.md for full prerequisites (only needed to rebuild the native library from source).

Install & configure

  • Install ai-core.cgp via the CoGo Plugin Manager before the plugins that depend on it, then restart the IDE.
  • AI Core has no settings screen of its own — select a .gguf model (Local) or enter a Gemini API key (Gemini) from AI Assistant → AI Settings. AI Core reads that configuration at request time.
