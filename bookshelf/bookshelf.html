1. Executive Overview

Bookshelf bundles a small curated library of programming reference textbooks — covering C, C++, Java, Kotlin, Android, and Pebble templates — and installs them into Code on the Go's existing in-IDE Bookshelf help page. Before the plugin is installed, the Bookshelf entries display a placeholder PDF; after installation, the same entries open the real book. Uninstalling restores the placeholders.

The plugin is read-only on the network and ships every book bundled inside the .cgp, so it works offline once installed. No accounts, no downloads, no telemetry.

2. Core Functionality

On install, Bookshelf swaps eight placeholder PDF rows in CoGo's documentation database for the corresponding compressed textbook. On uninstall, the rows revert to the placeholder. The books shipped today are:

Each book is delivered as a brotli-compressed PDF (.pdf.br) to keep the bundled .cgp small; CoGo's documentation viewer decompresses on read.

3. Technical Architecture

Bookshelf implements two interfaces from plugin-api: IPlugin for lifecycle and DocumentationExtension for the install/uninstall hooks.

Installation logic lives in onDocumentationInstall(). It opens the IDE's documentation.db SQLite database via SQLiteDatabase.openDatabase(…, OPEN_READWRITE), enumerates assets/books/*.pdf.br from the plugin's own asset bundle, and for each book:

  1. Reads the pre-compressed bytes through ResourceManager.openPluginAsset(…).
  2. Deletes any rows at the target path (and its <path>-N fragments) from the Content table.
  3. Inserts the brotli-compressed payload, chunked into 1 MiB rows when larger than the chunk size, with content type ID 14 (which the host schema reserves for application/pdf with brotli compression).

The entire batch runs inside a single transaction; if anything fails the transaction is rolled back so the documentation database is never left half-mutated.

Uninstall (onDocumentationUninstall()) queries the same table for rows under bookshelf/org.appdevforall.bookshelfplugin/%.pdf, deletes them, and recreates each one by copying the host's t/textbookPlaceholder.pdf row (and its fragments) back into the original path. This relies on the placeholder row already existing in the schema, which is the case in CoGo's stock documentation.db.

getTier3DocsAssetPath() returns null on purpose: the host's default Tier-3 asset insert would race with Bookshelf's manual database writes, so the plugin opts out of the automated path.

4. Usage

  1. Open Code on the Go's Plugin Manager.
  2. Install the bookshelf.cgp artifact (either from a local file or from the plugin catalog if it has been published there).
  3. Open Help → Bookshelf in the IDE.
  4. Select any of the eight titles — each opens the real PDF directly inside the help viewer.

To revert, uninstall the plugin from the same Plugin Manager view; the Bookshelf entries return to their placeholder PDF.

5. Key Benefits

  • Offline-first. No network access is required at install or read time — everything ships inside the .cgp.
  • Curated. Eight widely-cited free programming textbooks, hand-picked for relevance to the languages most CoGo users write.
  • Non-destructive. Uninstall restores the host's placeholder PDFs, leaving no orphan database state behind.
  • Compact. All eight books fit into roughly 24 MiB of brotli-compressed payload, decompressed on demand by the host viewer.
  • Permissioned narrowly. Requests only filesystem.read, filesystem.write, and ide.environment.write; no network or shell access.
