1. Executive Overview

Client Time Tracker is a Code on the Go plugin for contractors who bill by the hour. It adds a Client Time Tracker editor tab, plus a matching side-menu entry, that records billable coding time automatically from editor activity against the projects you register. It turns those sessions into invoices in PDF, Excel, or CSV.

Each tracked project carries its own client details, hourly rate, currency, and tax rate. As you edit, save, and build inside a registered project, the plugin opens and heartbeats a work session in the background. An idle timer closes the session when you stop.

All state lives in the plugin's own standalone Room database (contractor.db) under its private data directory. It never reads or writes the IDE's databases. The plugin implements three extension interfaces (IPlugin, UIExtension, EditorTabExtension), consumes four host services, and subscribes to the host's editor and project events over EventBus.

2. Core Functionality

The Client Time Tracker tab

The plugin contributes one editor tab, Client Time Tracker, hosted by ContractorMainFragment. Inside it, a Material TabLayout switches between three screens, showing the active screen and hiding the others (no ViewPager). An overflow button opens Settings as a bottom sheet:

ScreenPurpose
ProjectsRegister, edit, and archive tracked projects (client, rate, currency, tax).
SessionsReview tracked work sessions; edit or delete individual rows.
InvoicePick a project and date range, preview the computed total, and export.

Registering a project

A tracked project binds a project root path on disk to billing settings. Once registered, editor activity under that root counts toward that project. Fields: display name, client name, email, address, hourly rate, currency (ISO code, default USD), tax rate percent, and notes. Rows are unique by rootPath. Projects can be archived rather than deleted.

Automatic session tracking

SessionTracker drives tracking. It determines the active project by asking the host which project is open (IdeProjectService.getCurrentProject()) and matching its root against the registered projects. Only activity whose file path falls under the active project's root is counted.

A work session opens on the first activity and stays alive through heartbeats:

SignalSource eventEffect
EditDocumentChangeEventHeartbeat, increment edit count
SaveDocumentSaveEventHeartbeat, increment save count
Open / CloseDocumentOpenEvent / DocumentCloseEventHeartbeat
BuildBuildStatusListener.onBuildStarted/Finished/FailedOpen or keep the session alive
Project switchProjectInitializedEvent, OnResumeEventRe-evaluate the active project
BackgroundOnPauseEventFlush pending counts to the DB

To limit writes, heartbeats are persisted at most once every 60s (HEARTBEAT_PERSIST_INTERVAL_MS). Edit and save counts accumulate in memory and flush on the interval, on pause, or on close.

Idle close and crash recovery

An idle loop ticks every 30s. If no activity arrives for the configured idle threshold (Settings, default 5 minutes), the active session closes at its last heartbeat, so an invoice never bills time spent away from the keyboard. On startup, recoverOrphanedSessions() closes sessions left open by a process that was killed without a clean shutdown, capping them at their last persisted heartbeat.

Invoice generation

GenerateInvoiceUseCase turns a project and date range into an InvoiceData document:

  1. Load sessions in range; drop zero-duration sessions.
  2. Merge sessions separated by less than the configured gap (sessionMergeGapMinutes) so brief pauses don't fragment a day.
  3. Group by calendar day; each day becomes a line item of hours × rate.
  4. Sum to a subtotal, apply the project's tax percent, and total.
  5. Fold in business info (from Settings) and client info (from the project).

If no billable sessions fall in the range, the use case returns NoBillableSessions and the UI shows an empty state instead of an empty invoice.

Export formats

Every exporter implements a common InvoiceExporter interface (export(data, outFile): File). Currency and dates are formatted once in InvoiceFormatting for consistency across formats.

FormatExporterImplementation
PDFPdfInvoiceExporterAndroid PdfDocument canvas rendering
Excel (.xlsx)XlsxInvoiceExporterorg.dhatim:fastexcel
CSVCsvInvoiceExporterDirect text writer

The generated file is handed to the user through the host IDE's FileProvider (see Integration Points) so it can be shared or opened by other apps.

Data model

The database is self-contained: three tables, the latter two foreign-keyed to projects with ON DELETE CASCADE.

TableEntityHolds
tracked_projectsTrackedProjectEntityRoot path, client, rate, currency, tax, archived flag
work_sessionsWorkSessionEntityProject id, start/end, last heartbeat, edit/save counts
invoicesInvoiceEntityProject id, number, period, totals, currency, exported files

Room writes use explicit @Query, not @Insert

Plugins run on the host's Room runtime via a parent-first DexClassLoader, so a plugin's generated entity adapters subclass the host's EntityInsertAdapter. In that topology the adapter's createQuery() can be mis-dispatched across the classloader boundary to a host DAO's statement. To avoid it, every write in this plugin is an explicit inline statement: 

@Query("INSERT INTO tracked_projects (id, rootPath, …) VALUES (:id, :rootPath, …)")
suspend fun insertRow(id: String, rootPath: String, …)

suspend fun insert(project: TrackedProjectEntity) = insertRow(project.id, project.rootPath, …)

Data flow

editor activity (edit / save / open / close / build) | | host EventBus + IdeBuildService.BuildStatusListener v SessionTracker ├─ active project? (IdeProjectService.getCurrentProject vs registered roots) ├─ ensureActiveSession() ──> open work_sessions row ├─ handleHeartbeat() ──> throttle, accumulate edit/save counts └─ idle loop / onPause ──> flush + close | v contractor.db (Room: tracked_projects · work_sessions · invoices) | | Invoice screen: project + date range v GenerateInvoiceUseCase (merge, group-by-day, rate × hours, tax, total) | v InvoiceExporter (PDF | XLSX | CSV) ──> host FileProvider ──> share / open

4. Integration Points

Client Time Tracker implements three extension interfaces, consumes four host services, subscribes to host events, and borrows the host's FileProvider.

4.1 Plugin lifecycle (IPlugin)

initialize() builds the object graph through ContractorServiceLocator.init(). activate() starts the SessionTracker. deactivate() and dispose() stop the tracker and close the database. Every callback is wrapped in runCatching, so a setup error degrades gracefully instead of crashing the host.

4.2 Editor tab and side menu (EditorTabExtension, UIExtension)

getMainEditorTabs() contributes the Client Time Tracker tab; its fragment factory returns a fresh ContractorMainFragment. getSideMenuItems() adds a navigation entry that opens the same tab via IdeEditorTabService.selectPluginTab().

4.3 Host services consumed

ServiceUsed for
IdeEnvironmentServicegetPluginDataDirectory(): where contractor.db lives
IdeProjectServicegetCurrentProject(): resolve the active project root
IdeBuildServiceaddBuildStatusListener(): count build time as activity
IdeEditorTabServiceOpen or select the plugin tab from the side menu

4.4 Host events (eventbus-events.jar)

SessionTracker registers with the host's EventBus.getDefault() and subscribes to com.itsaky.androidide.eventbus.events.editor.* and events.project.* events. These classes are not part of plugin-api.jar, so this plugin needs a third compile-only jar: 

compileOnly(files("../libs/plugin-api.jar"))
compileOnly(files("../libs/eventbus-events.jar"))
compileOnly("org.greenrobot:eventbus:3.3.1")
Note: any plugin that subscribes to host events over EventBus needs eventbus-events.jar in the repo-root libs/ folder. It is vendored from CodeOnTheGo's eventbus-events module and used compileOnly. The IDE provides the classes at runtime.

4.5 Sharing exports: the FileProvider hop

A dex-loaded plugin cannot register its own FileProvider, so exported invoices are shared through the host's existing authority (${HOST_APPLICATION_ID}.providers.fileprovider), resolved at runtime in InvoiceSuccessSheet. The host's FileProvider paths whitelist the relevant directory and grant temporary read access to the target app.

4.6 Plugin-scoped fragment restoration

The host activity is recreated on a theme (uiMode) change, and FragmentManager restores nested fragments by class name using the host classloader. ContractorMainFragment installs a PluginChildFragmentFactory on its childFragmentManager (before super.onCreate()) that loads plugin fragments through the plugin's DexClassLoader. Without it, restoring an open bottom sheet after a theme toggle throws ClassNotFoundException.

4.7 Permissions

<meta-data android:name="plugin.permissions"
           android:value="filesystem.read,filesystem.write,ide.environment.write" />

Filesystem permissions back invoice export to disk; ide.environment.write backs the plugin's private data directory.

5. Deployment & Usage

Building

cd client-time-tracker
./gradlew assemblePlugin

Produces client-time-tracker/build/plugin/client-time-tracker.cgp, the bundle you sideload into Code on the Go.

Before building: this plugin needs ../libs/eventbus-events.jar in addition to the usual plugin-api.jar and gradle-plugin.jar. See §4.4 and the repo-root README's libs/ notes.

Installation

  1. Open Preferences → Plugin Manager → +.
  2. Select the client-time-tracker.cgp file.
  3. The IDE discovers ContractorPlugin via manifest metadata and activates it.
  4. Open the Client Time Tracker tab from the editor tab bar or the side menu.

Using the plugin

  1. Projects → register a project: point it at a project root and set the client, hourly rate, currency, and tax rate.
  2. Work normally. While that project is open in the IDE, editing, saving, and building accrue time to it.
  3. Sessions → review tracked sessions; edit or delete any row that needs adjusting.
  4. Invoice → choose the project and a date range, review the total, and export to PDF, Excel, or CSV.
  5. Tune the idle threshold, session-merge gap, default tax and currency, and business details under the overflow menu → Settings.

6. Key Benefits

  • Hands-off billable tracking. Time accrues from editor and build activity, with idle auto-close and crash recovery, so invoices reflect work done.
  • Invoices in three formats. One InvoiceData model, three exporters (PDF, Excel via fastexcel, CSV), shared through the host FileProvider.
  • Standalone storage. A private Room database that never touches the IDE's data, a model for plugins that need persistence.
  • Event- and service-driven reference. Demonstrates host EventBus subscription, BuildStatusListener, and consuming four IDE services together in one plugin.
  • UI patterns. A multi-screen tabbed fragment, MVI view models, ViewBinding, and a plugin-scoped child FragmentFactory that survives theme-toggle recreation.
  • Documented Room-on-host gotcha. Shows the inline @Query write pattern that sidesteps cross-classloader entity-adapter mis-dispatch, useful for any Room-backed plugin.
