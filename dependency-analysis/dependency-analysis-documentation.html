1. What it does

Dependency Analysis runs the autonomousapps Dependency Analysis Gradle Plugin (DAGP) against the open project and reports dependencies that are unused, used-but-undeclared (transitive), or declared on the wrong configuration. A single Fix dependencies action applies DAGP's recommended edits and re-analyzes.

2. The three kinds of advice

CategoryMeaningRecommended action
UnusedDeclared but not referenced by code.Remove the declaration.
Undeclared transitiveUsed directly but only present transitively.Declare it explicitly.
MisconfiguredOn the wrong configuration (e.g. api vs implementation).Move it.

3. Running an analysis

Open the Dependencies tab in the editor bottom sheet and tap Analyze dependencies. The plugin runs Gradle's :buildHealth task with a generated init script that applies DAGP on the fly — your build files are not touched. Advice is read from:

build/reports/dependency-analysis/build-health-report.json

4. The one-tap fix

Tapping Fix dependencies confirms, then delegates to DAGP's :fixDependencies task, which rewrites your build.gradle(.kts) files in place. The plugin then re-runs the analysis and shows a post-apply summary of how many dependencies were removed, added, and reconfigured. No build-file diff is synthesized — Gradle performs the edits and the re-analysis confirms the result.

Kotlin DSL rewriting is reliable; Groovy DSL rewriting is best-effort. If a build.gradle is open in the editor during a fix, reload it to see the on-disk edits.

5. Permissions & bundling

The plugin declares exactly system.commands (to run the Gradle tasks) and filesystem.read (to read the report). It does not request network access: DAGP and its dependencies are bundled with the plugin so analysis works offline. Bundled third-party libraries are listed in the plugin's THIRD_PARTY_LICENSES asset.
