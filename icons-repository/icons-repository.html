IconsRepository-Plugin

A Code on the Go (CoGo) plugin that adds vector drawable icons to Android projects directly from inside the IDE.

Executive overview

IconsRepository-Plugin is a sidebar extension for the Code on the Go IDE that lets developers browse a bundled library of Material-style icons and import any of them into the current Android project as a Vector Drawable XML. The plugin replaces the manual "find an SVG, convert it, copy it into res/drawable, fix the colors" loop with a single dialog: pick an icon, optionally pick a tint, hit Import. The result is committed-ready XML in the project's res/drawable directory with no external network calls and no SDK changes required.

The plugin targets Code on the Go versions 26.17 through 26.22 and runs on Android API 26 and above.

Originally created by OMAR HAIDAR (upstream repository) and included in this collection with attribution.

Core functionality

  • Bundled icon catalog. Ships with a ZIP-packed Material icon set as a plugin asset. No download required at runtime; works fully offline.
  • Searchable browser. A grid view with debounced search backed by a trie-based index so name queries stay responsive on full catalogs.
  • One-tap import. Selecting an icon opens an Import dialog where the developer chooses a drawable name and an optional tint. The plugin extracts the SVG path, generates a Vector Drawable XML, and writes it into the active app module's res/drawable directory.
  • Tint handling. Supports either a literal hex color or one of the IDE's dynamic Material colors (resolved via theme attributes), applied as android:tint on the generated vector.
  • Input validation. Drawable names and color values are validated before the file is written — invalid names or bad hex colors block the import.
  • Safe writes. Filenames are sanitized to [a-zA-Z0-9._-] before being joined with the destination directory; existing files are not overwritten.

Technical architecture

The plugin is a standard Code on the Go plugin module built with the com.itsaky.androidide.plugins.build Gradle plugin. The output of ./gradlew assemblePlugin is a .cgp bundle (signed APK with the plugin metadata) ready for the IDE's Plugin Manager.

Module layout

  • Main — implements IPlugin via BasePlugin; registers one sidebar item and one editor tab; owns the singleton IconRepository.
  • BasePlugin — common lifecycle plumbing on top of the plugin-api; exposes PluginContext, ServiceRegistry, and a logger to subclasses, and clears references in dispose().
  • repository.IconRepository — loads icons from the bundled material_icon.zip asset on a background thread, caches them in memory, and exposes an asynchronous load API. Provides shutdown() to terminate the executor on plugin disposal.
  • ui.fragments.MainFragment — the catalog UI: a grid RecyclerView driven by IconsAdapter, with a debounced search input and an import dialog.
  • ui.adapter.IconsAdapter — owns the search index, the SVG render executor pool, the main-thread handler, and the bitmap LruCache. All resources are released in onDetachedFromRecyclerView.
  • data.importer.SvgToVectorImporter — extracts SVG d path data, builds a Vector Drawable document with XmlPullParser, and writes it to the project's resource directory.
  • data.importer.AppPathResolver — resolves the active project's app/src/main/res/drawable through the IDE's IProject/IModule APIs (with a Gradle-plugin-project fallback).
  • data.validation — small validation framework (Validator, ValidationRule, IconNameRule, HexColorRule, RequiredRule) used by the import dialog.

Permissions

The plugin declares the following permissions in AndroidManifest.xml:

  • filesystem.read — for reading the bundled icon catalog.
  • filesystem.write — for writing the generated Vector Drawable XML.
  • project.structure — for resolving the active module's source sets and resource directories.

The plugin does not access the network, run system commands, change IDE settings, or ship native code.

Lifecycle and resource discipline

All long-lived resources are scoped to either the adapter or the plugin and are torn down explicitly. The search and render executors and the bitmap cache live on IconsAdapter and are shut down in onDetachedFromRecyclerView. The IconRepository executor is shut down in Main.dispose(), which also clears the Main.instance static reference. BasePlugin.dispose() clears PluginContext and IdeProjectService references so no IDE state survives plugin unload.

Usage

Installation

  1. Open Code on the Go.
  2. Open the Plugin Manager.
  3. Install IconsRepository-Plugin.

Importing an icon

  1. Open an Android project in the IDE.
  2. Tap the IconsRepository entry in the sidebar (or open the Icons Repository editor tab).
  3. Wait briefly for the icon catalog to load (a spinner is shown while the bundled archive is parsed).
  4. Use the search field to find an icon by name.
  5. Tap an icon to open the Import dialog.
  6. Choose a drawable name (defaults to ic_<icon-name> with dashes replaced by underscores).
  7. Optionally choose a tint: None, a custom hex color, or a dynamic Material color.
  8. Tap Import. The plugin writes <name>.xml into the app module's res/drawable directory.

Building from source

./gradlew assemblePlugin

The signed .cgp artifact is written to build/plugin/IconsRepository-Plugin.cgp. Use ./gradlew assemblePluginDebug for the debug variant.

Key benefits

  • Stays on-device. Everything ships in the plugin asset — no network access, no third-party CDN, no rate limits, no opt-in telemetry.
  • Vector-first output. Generates Android Vector Drawables, not PNGs, so imported icons scale cleanly and stay small in the APK.
  • Idiomatic placement. Files land in the active app module's standard res/drawable directory and follow the ic_* naming convention by default.
  • Lifecycle-clean. Threads, handlers, and caches are scoped to the adapter or plugin and torn down on disposal — no leaked executors or pinned IDE references.
  • Minimal permission surface. Only the three permissions the import flow actually exercises are requested; no network, system command, or native-code permissions.
  • Reproducible build. A single documented Gradle command (./gradlew assemblePlugin) produces the submission artifact on any clean machine with the Android SDK installed.
