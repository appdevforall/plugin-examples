1. Executive Overview

The Keystore Generator Plugin is a comprehensive Code on the Go (CoGo) plugin that enables Android developers to generate secure keystores directly within their development environment. This plugin eliminates the need for external tools and command-line operations, providing a seamless, integrated solution for Android app signing preparation.

2. Core Functionality

Keystore generation

Generates Android keystores (.jks files) with RSA 2048-bit encryption and SHA-256 signatures. Supports customizable certificate details, including organizational information.

Build integration

Automatically updates project build files (Gradle / Kotlin DSL) with signing configurations, streamlining the release-build process.

Security features

Implements the BouncyCastle cryptographic library for secure key generation, with comprehensive validation and error handling.

Multi-UI access

Provides multiple access points — main editor tabs, bottom sheet tabs, sidebar actions, and main-menu integration — for maximum developer convenience.

5. Security & Validation

Security features

  • RSA 2048-bit key generation with the SHA-256 signature algorithm.
  • X.509 certificates with a 25-year validity period.
  • Password validation with a minimum 6-character requirement.
  • Form validation with comprehensive error handling.
  • Build-status awareness — disables operations during active builds.

Validation rules

  • Keystore name must end with .jks or .keystore.
  • Passwords must be at least 6 characters.
  • Country code must be exactly 2 characters.
  • All required fields must be completed.
  • Real-time form validation with inline error display.
Plugin permissions: filesystem.read, filesystem.write, project.structure — required to read the project's existing build files, write the generated .jks, and patch the signing config back into the right module.

6. Build System Integration

The plugin automatically generates and inserts signing configurations into the project's build files.

Kotlin DSL (build.gradle.kts)

signingConfigs {
    create("release") {
        storeFile = file("release.jks")
        storePassword = "your_password"
        keyAlias = "release"
        keyPassword = "your_key_password"
    }
}

Groovy (build.gradle)

signingConfigs {
    release {
        storeFile file('release.jks')
        storePassword 'your_password'
        keyAlias 'release'
        keyPassword 'your_key_password'
    }
}

7. Deployment & Usage

Plugin packaging

AttributeValue
Build targetAndroid application (APK)

DistributionRelease and debug variants available

Installation

  1. Build the plugin: ./gradlew assemblePlugin
  2. Install the resulting .cgp file in Code on the Go.
  3. Access via the sidebar, main menu, or editor tabs.
  4. Configure keystore details and generate.

Developer experience

  • User interface: clean, intuitive form with real-time validation, progress indicators, and clear error messaging.
  • Performance: asynchronous keystore generation using Kotlin coroutines to prevent UI blocking during cryptographic operations.
  • Documentation: integrated help system with security best practices, usage guides, and contextual tooltips throughout the interface.
  • Build awareness: intelligent build-status monitoring that disables operations during active builds to prevent conflicts.
