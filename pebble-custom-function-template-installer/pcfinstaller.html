PCFInstaller

A Code on the Go (CoGo) plugin that installs two project templates for authoring and demonstrating Pebble Custom Functions.

Executive overview

PCFInstaller registers two new project templates with the Code on the Go IDE: PCFBuilder for authoring a Pebble Custom Function library (the extensions.jar artifact that Pebble loads), and PCFExample for consuming the resulting custom functions from a sample app. After installation the templates appear on the IDE's New Project screen, so a developer can scaffold a complete Pebble Custom Function workflow without copying files by hand.

The plugin targets Code on the Go versions 26.22 through 26.23. It requires only filesystem.read and filesystem.write permissions and performs no network calls at runtime.

Core functionality

  • PCFBuilder template. Generates a Gradle project that builds extensions.jar, the binary that ships one or more Pebble Custom Function classes. Each function lives in its own class so Pebble templates can reference it by name.
  • PCFExample template. Generates a sample Android application that consumes the custom functions produced by PCFBuilder, providing a runnable end-to-end demonstration.
  • Optional example code. Both templates expose an "Include Sample Functions" checkbox. When enabled, four reference functions (HelloUserFunction, ReverseFunction, QueryURLFunction, QueryDbFunction) are scaffolded into the project.
  • Template parameters. Author, description, package name, minimum SDK, and language (Java/Kotlin) are all collected from the standard New Project dialog.
  • Clean deactivation. Disabling the plugin removes the staged template directories it owns, so re-enabling produces a clean install.

Technical architecture

PCFInstaller is a standard Code on the Go plugin module built with the com.itsaky.androidide.plugins.build Gradle plugin. The output of ./gradlew assemblePlugin is pcfinstaller.cgp, the bundle consumed by the IDE's Plugin Manager.

At runtime the main class com.example.pcfinstaller.PCFInstaller implements IPlugin and uses three IDE services obtained from PluginContext.services:

  • IdeTemplateService — builds and registers the two templates via createTemplateBuilder(...) → addStaticFromAssets(...) → build(...) → registerTemplate(...).
  • IdeEnvironmentService — resolves the IDE's Android home directory so the deactivation hook knows where to clean up.
  • IdeFileService — performs the actual directory removal on deactivation.

Template payloads ship inside the plugin's own APK assets under assets/templates/PCFBuilder/ and assets/templates/PCFExample/. The plugin enumerates these payloads via the Android AssetManager at activation time and copies each file into the template through the IdeTemplateService builder — no external filesystem path is read.

Lifecycle:

  • initialize(PluginContext) caches the context and resolves the IDE services.
  • activate() registers both templates.
  • deactivate() removes the staged template directories under the IDE's Android home.
  • dispose() cancels the coroutine scope and nulls the cached template-service reference.

Usage

  1. Open the Code on the Go app on your Android device.
  2. Open Preferences → Plugin Manager.
  3. Either choose Install from URL (download icon, upper right) and paste the location of pcfinstaller.cgp, or use the + button (lower right) to search the plugin catalog for PCFTemplateInstallerPlugin.
  4. Restart the IDE so the templates register.
  5. From the New Project screen, pick either PCFBuilder (to author a Pebble Custom Function library) or PCFExample (to scaffold a consumer of those functions).
  6. Fill in Author, Description, and the standard project options. Leave Include Sample Functions checked to get the four reference implementations.
  7. To remove the templates, disable the plugin in the Plugin Manager — the staged directories are cleaned up automatically.

Key benefits

  • Zero-friction scaffolding. Removes the manual setup that previously stood between a developer and a working Pebble Custom Function project.
  • End-to-end coverage. Ships both the producer (extensions.jar builder) and the consumer (sample app), so the full Pebble flow is exercisable from a fresh device.
  • Self-contained. No network access required at runtime; all template assets are bundled in the plugin.
  • Minimal permissions. Requests only filesystem.read and filesystem.write; no reflection, no native code, no IDE-internal access.
  • Reversible. Deactivation removes the templates so the IDE returns to its original state.
