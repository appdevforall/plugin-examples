RainbowOnTheGo

RainbowOnTheGo colors matching parentheses, brackets, and braces by how deeply they nest, so a delimiter and its match share a color. It works in every language and adds color only — your normal syntax highlighting is untouched.

What you'll see

Each nesting level gets the next color, cycling through six:

  • Depth 0 — red
  • Depth 1 — orange
  • Depth 2 — yellow
  • Depth 3 — green
  • Depth 4 — blue
  • Depth 5 — purple

At depth 6 the colors start over at red. A closing bracket always matches the color of the opening bracket it pairs with.

Light and dark themes

The plugin ships two palettes. On a light editor theme it uses deeper, saturated colors that stay legible on white; on a dark theme it uses brighter, pastel colors. The editor picks the palette that matches your current theme and repaints when you switch.

Strings and comments

Brackets inside string literals and comments are skipped, so a parenthesis in "hello (world)" or in a // comment (here) keeps its normal color and does not affect nesting depth.

Turning it off

Disable RainbowOnTheGo in the Plugin Manager to return all brackets to the editor's normal color. The change applies immediately to open editors.

How it's built

RainbowOnTheGo implements EditorDecorationProvider, a generic Code On The Go plugin API. Its decorate method scans a region of text for bracket characters, tracks the running nesting depth, skips brackets inside strings and comments, and returns a foreground-color span per bracket. The editor merges those spans on top of the normal highlighting — it never replaces it, and the IDE itself knows nothing about brackets. The same generic hook could power other decorations too.
