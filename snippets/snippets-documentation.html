1. Executive Overview

Custom Snippets is a Code on the Go plugin that lets developers define, manage, and expand their own code snippets directly inside the editor. Snippets are stored per-project as JSON, contributed to the IDE's completion system via SnippetExtension, and managed through a dedicated editor tab with a full CRUD interface.

The plugin ships a built-in set of four Java defaults (such as sout, logi, trycatch, and singleton) that bootstrap automatically on first use, so there is something to try immediately. After that, developers can add, edit, and delete snippets through the UI or by editing the JSON file directly — both paths converge on the same parser and the IDE picks up changes without a restart.

The plugin spans three Kotlin source files and three XML layouts (~530 lines of code total), requires only filesystem.read and filesystem.write permissions, and demonstrates several advanced plugin capabilities: snippet contributions, editor tabs, side-menu navigation, fragment-based UI, and live service registry interaction.

2. Core Functionality

Snippet contributions

The plugin implements SnippetExtension to feed prefix-triggered expansions into the IDE's completion popup. When a user types a prefix like sout and accepts the completion, the body is expanded in place. Placeholders follow the LSP tab-stop convention:

See also TextMate Snippets.

SyntaxMeaning
$1, $2, ...Tab stops visited in order
${1:default}Tab stop with a default value
$0Final caret position

Snippet scopes

Every snippet has a scope that controls where in the source file it is offered as a completion. The four scopes are:

ScopeWhere it expandsExample use
global Anywhere in the file File-header comments, license blocks
top-level At the top level of the file, outside any class or method Import groups, class declarations
member Inside a class body, at member level Field declarations, method stubs, the singleton default
local Inside a method or block Statements like sout, logi, trycatch

Scoping keeps the completion popup relevant — a singleton pattern won't appear inside a method body, and a sout call won't clutter completions at class level.

Snippet manager UI

A "Snippets" editor tab hosts SnippetManagerFragment, which provides a scrollable card-based list of all snippets in the current project. Each card shows the prefix, description, language/scope tags, and a body preview. Users can:

  • Add a new snippet via the "+ Add" button, which opens a Material dialog with fields for prefix, description, language, scope, and body.
  • Edit an existing snippet by tapping its card.
  • Delete a snippet via the trash-can icon, with a confirmation dialog.

All changes are saved immediately to disk, the in-memory cache is invalidated, and IdeSnippetService.refreshSnippets() is called so the editor picks up changes without a restart.

Per-project storage

Snippets live at .cg/snippets.json in the project root. The JSON schema is: 

{
  "snippets": [
    {
      "language": "java",
      "scope": "local",
      "prefix": "sout",
      "description": "System.out.println",
      "body": ["System.out.println($1);$0"]
    }
  ]
}
FieldPurpose
languageTarget language id (java, kotlin, xml, groovy, json)
scopeWhere the expansion applies (global, local, member, top-level)
prefixTrigger text typed in the editor
descriptionLabel shown in the completion popup
bodyLines of the expansion, joined with \n

Default snippets (bootstrap)

If .cg/snippets.json does not exist when the plugin first reads it, four Java snippets are written automatically:

PrefixScopeDescription
soutlocalSystem.out.println
logilocalLog.i with TAG
trycatchlocalTry-catch block
singletonmemberSingleton instance pattern

File-change detection

getSnippetContributions() compares lastModified() on the snippets file against a cached timestamp. If the file has been edited on disk (by hand or by the manager UI), the cache is invalidated and the file is re-parsed. This means hand-editing the JSON and using the manager UI are fully interchangeable.

Snippet loading and caching

override fun getSnippetContributions(): List<SnippetContribution> {
    val projectRoot = getProjectRoot()
        ?: return cachedContributions ?: emptyList()
    val snippetsFile = File(projectRoot, SNIPPETS_PATH)

    if (!snippetsFile.exists()) {
        bootstrap(snippetsFile)
    }

    val currentMtime = if (snippetsFile.exists())
        snippetsFile.lastModified() else 0L
    if (currentMtime != snippetsLastModified
            || cachedContributions == null) {
        cachedContributions = loadSnippets(snippetsFile)
        snippetsLastModified = currentMtime
    }

    return cachedContributions ?: emptyList()
}

The method is called by the IDE whenever it needs snippet completions. It bootstraps defaults if the file is missing, then uses a lastModified() comparison to avoid re-parsing on every call.

Manager UI save path

private fun save() {
    val file = SnippetsPlugin.instance?.getSnippetsFile() ?: return
    SnippetsConfigParser.write(file, SnippetsConfig(snippets.toList()))
    SnippetsPlugin.instance?.invalidateCache()
    SnippetsPlugin.instance?.refreshRegistry()
}

After writing JSON to disk, the fragment invalidates the plugin's in-memory cache and calls refreshSnippets() on IdeSnippetService so the editor's completion list updates immediately.

4. Integration Points

Snippets touches the host IDE through four extension interfaces and three services. It does not modify any main-app source files — all integration happens at runtime through the plugin framework.

4.1 Plugin lifecycle (IPlugin)

SnippetsPlugin implements IPlugin. The manifest declares the entry point: 

<meta-data android:name="plugin.main_class"
           android:value="com.Code on the Go.snippets.SnippetsPlugin" />

initialize() stores the PluginContext and sets a static instance reference (used by the fragment). deactivate() clears the snippet cache. dispose() nulls out the context and instance.

4.2 Snippet extension (SnippetExtension)

The IDE calls getSnippetContributions() to collect snippets for the completion popup. The plugin returns a list of SnippetContribution objects, each mapping a prefix to an expansion body with tab stops. The result is cached and invalidated by file-modification timestamp.

4.3 Editor tab (EditorTabExtension)

getMainEditorTabs() contributes a closeable, non-persistent tab titled "Snippets" with a custom icon: 

EditorTabItem(
    id = TAB_ID,
    title = "Snippets",
    icon = R.drawable.ic_snippet,
    fragmentFactory = { SnippetManagerFragment() },
    isCloseable = true,
    isPersistent = false,
    order = 100
)

4.4 Side-menu navigation (UIExtension)

getSideMenuItems() adds a "Manage Snippets" entry under the tools group. Tapping it calls IdeEditorTabService.selectPluginTab(TAB_ID) to open or focus the manager tab.

4.5 Services consumed

ServiceUsage
IdeProjectService getCurrentProject().rootDir — locates the project root to find .cg/snippets.json
IdeEditorTabService selectPluginTab() — opens the Snippets manager tab from the side menu
IdeSnippetService refreshSnippets(pluginId) — tells the IDE to re-read snippet contributions after a save

4.6 Fragment infrastructure

SnippetManagerFragment uses PluginFragmentHelper to obtain a plugin-scoped LayoutInflater (required for resolving resources bundled inside the .cgp package) and to access the service registry from within the fragment.

4.7 Permissions

The manifest declares filesystem.read and filesystem.write, which are needed to read and write .cg/snippets.json in the project directory.

5. Deployment & Usage

Building

./gradlew :snippets:assemblePluginDebug

This produces a .cgp plugin package under snippets/build/outputs/.

Installation

  1. Open Code on the Go's plugin manager.
  2. Import the .cgp file.
  3. The IDE discovers SnippetsPlugin via the manifest metadata and activates it automatically.
  4. Open any project — four default Java snippets are bootstrapped into .cg/snippets.json on first access.

Using snippets in the editor

  1. Start typing a prefix (e.g. sout) in a Java file.
  2. The completion popup shows the snippet with its description.
  3. Accept the completion — the body is inserted with tab stops for quick editing.

Managing snippets

  • Via the UI: Open Tools > Manage Snippets from the side menu, or switch to the "Snippets" editor tab.
  • Via the file: Edit .cg/snippets.json directly. Changes are detected by lastModified() on the next completion request.

6. Key Benefits

  • Faster coding on mobile. Typing boilerplate on a phone keyboard is painful. Snippets turn a short prefix into a full code block with tab-stop navigation, saving keystrokes where they cost the most.
  • Per-project, version-controllable. Snippets are stored in .cg/snippets.json inside the project root, so they can be committed to source control and shared across a team.
  • Dual editing paths. The graphical manager UI and direct JSON editing are fully interchangeable — the file-modification timestamp ensures both paths stay in sync without restarts.
  • Live registry refresh. After saving, the plugin calls IdeSnippetService.refreshSnippets() so the editor's completion list updates immediately — no need to close and reopen files.
  • Bootstrapped defaults. Four practical Java snippets are created on first use, so the plugin is useful out of the box and serves as examples for writing custom entries.
  • Rich reference implementation. The plugin demonstrates four extension interfaces (IPlugin, SnippetExtension, EditorTabExtension, UIExtension), three IDE services, fragment-based UI with PluginFragmentHelper, Material Design dialogs, and per-project file I/O — making it a comprehensive template for building more complex plugins.
